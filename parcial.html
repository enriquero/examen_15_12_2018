Proyecto de Software 2018 - Parcial 3era Fecha - 15/12/2018

Tema 2

Se requiere desarrollar un prototipo simplificado de una aplicación similar a Twitter, que permita publicar mensajes de no más de 150 caracteres.  Los mensajes podrán ser visualizados por cualquier usuario, si el mensaje es público, caso contrario, sólo los/as usuarios/as seguidores podrán ver el mensaje.  En todos los casos siempre deberán iniciar sesión para acceder a los mensajes.

Para tal fin, se debe implementar la siguiente funcionalidad:

  • Login de usuario: se deberá desarrollar el login y logout de usuario, validando en el cliente y en el servidor que los datos ingresados no estén blanco y en el servidor que las credenciales sean las correctas.

Agregue un link o botón en el home para acceder a esta funcionalidad.

  • Carga de un mensaje: en esta sección se permite cargar un mensaje completando los datos con la siguiente información:

  • Mensaje* (TEXT): validar en cliente y servidor que no esté en blanco. Máximo 150 caracteres.
  • Fecha (DATE): fecha en la que se carga el mensaje.
  • Público*: indica si el mensaje es público o no.
  • Autor: usuario logueado en el sistema que realizó la carga del mensaje.

Nota: Para realizar la carga se requiere iniciar sesión. Agregue un link o botón en el home para acceder a esta funcionalidad.

  • Listado de mensajes: una vez que el usuario haya ingresado a la aplicación (debe iniciar sesión), se podrán listar los mensajes ordenadas por fecha descendente.  Tenga en cuenta que el usuario podrá ver los mensajes públicos o los mensajes de los usuarios a quien sigue.

Agregue un link o botón en el home para acceder a esta funcionalidad.

Modelo simplificado de la base de datos

Aclaraciones:

  • Cada alumno/a dispondrá de un dump de la base de datos que deberá importar para realizar el parcial.  Para tal fin, se le proporcionará el usuario y contraseña para conectarse a MySQL.
  • Credenciales de la VM (son sudoer)
  • usuario: alumno
  • clave: alumno
  • Credenciales de MySQL
  • usuario: root
  • clave: alumno
  • Debe respetar MVC y controlar los datos enviados desde el cliente al servidor.
  • Se debe implementar utilizando PDO o Doctrine y respetar las sesiones planteadas.
  • La vista debe ser implementada mediante Twig.
  • Los datos ingresados por el usuario no deben ser susceptibles a SQL Injection ni XSS.
  • Realice las aclaraciones que crea necesarias tanto en el código con comentarios como en un archivo de texto.
  • Verifique haber subido el código completo del parcial, se corregirán los archivos que se encuentren en el repositorio de Git.
